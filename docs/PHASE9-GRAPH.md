# Phase 9 (R4) — The provenance graph (Obsidian-style)

The Story view is linear. A graph shows what a list hides: **fan-out** (one prompt exploding into many steps), **shared-entity relationships** (a file touched by several steps is a high-degree node), and a **risk chain as a drawn path** (the exfil combo `secret → send → host` rendered in red — the postable screenshot). R4 adds a third **Graph** tab beside Timeline and Story.

## The load-bearing rule
Entirely a **read-time interpretation** — no capture, no schema change, nothing hashed, `verify()` untouched. `buildGraph` is a pure projection of the existing story + risk combos; `sessionGraph` only reads. This is the cleanest of the four systems: it invents no new facts, just a new lens on the ones already recorded.

## The projection (`src/graph.ts`)
`buildGraph(story, combos, promptId?) → { nodes, edges }` at two resolutions:
- **Session overview** (default): `prompt` · `file` · `commit` · risk nodes; steps are aggregated (a prompt links straight to the files it changed). Stays readable for a big session (~76 nodes for a 51-turn session, not ~1000).
- **Single turn** (`?prompt=<id>`): `prompt` · every `step` · `file` · `commit` — the full causal detail of one turn.

**Node types** `prompt · step · file · commit · host · secret · mcp`, each with a drill-in `seq`. **Edge types** `caused · changed · committed · spawned · read · sent · combo`. A fired **exfil-chain** combo adds a `secret` node (at the secret-touch seq), a `host` node, and a red **combo** path — anchored to the step nodes in the detailed view or to the owning prompt in the overview (so the risk shows up at both resolutions, even when the combo evidence carries no structured host). Subagent steps nest under their spawning `Task` via a `spawned` edge. Node `degree` is computed so the UI can size by connectedness.

**Anchoring is exact, never fabricated.** A combo is drawn only where it actually belongs. Each combo seq resolves through `seqToStep` (a step's own seq *and* its `post_seq` — real combos usually cite the POST/output seq, e.g. a redaction-detected secret-touch) and falls back to the owning `seqToPrompt`; if **neither** the antecedent nor the consequent resolves inside the current (sub)graph, the combo is **skipped entirely**. This is what stops a single-turn view from painting another turn's exfil chain as a floating `secret → host` path, keeps a poisoned-`mcp` node connected to the step that used it rather than orphaned, and prevents a `post_seq` combo from being mis-blamed on the prompt instead of the step that fired it.

Wired: `read-api.sessionGraph` (story + `session_risk` combos → `buildGraph`); daemon route `GET /api/session/:id/graph` with optional `?prompt=`.

## The UI (`src/ui-page.ts`)
A **hand-rolled force-directed canvas** — zero deps (the CSP forbids CDN libs, so no d3/Cytoscape). A simple repulsion + spring + center-gravity simulation runs on `requestAnimationFrame` (~240 frames then freezes); under `prefers-reduced-motion` it computes a static layout synchronously and draws once. Nodes are coloured by type and **sized by degree**; risk nodes/edges are the accent red, the combo path dashed. **Pan** (drag), **zoom** (wheel, around the cursor), **hover** highlights a node's neighbourhood (dims the rest), and **clicking a node opens its dossier** below the canvas (reusing the timeline's `insertDetail`). A **focus** selector switches between the whole session and a single turn's subgraph. A trivial (≤1-node) graph shows a hint to use the Story instead. The rAF loop and the window-resize listener are torn down on every view/session switch (no leak, no draw-after-teardown).

## Files
`src/graph.ts` (new) · `src/read-api.ts` (`sessionGraph`) · `src/daemon.ts` (`/graph` route) · `src/ui-page.ts` (Graph tab + canvas renderer) · `test/graph.test.js` (17).

## Adversarial pass (post-commit fix)
A dynamic verification workflow (parallel projection-lens agents) caught three real projection defects *after* the initial R4 commit, all fixed here: (1) **cross-turn combo leak** — a single-turn view drew an unrelated turn's `secret → host` exfil chain as a floating red path, because detailed mode filtered turns but not combos; (2) **`post_seq` mis-anchor** — step nodes are keyed by PRE seq while combos cite the POST seq, so the risk attached to the prompt instead of the firing step; (3) **orphaned `mcp` node** — the tool-poisoning node floated with no incident edge. The fix threads a `seqToStep` map (PRE + POST → step id) and skips any combo whose antecedent and consequent both resolve to nothing in the current subgraph.

## Optimization pass — sharp, hybrid-layout, useable (the "looks like shit" fix)
The first cut worked but read as low-quality: **blurry on retina** (the canvas was never DPR-scaled), a **faint-grey hairball** (edges at α 0.11), **no way to drag a node** (drag only panned), and a sim that **froze after 240 frames** mid-tangle. This pass rebuilds the renderer to the plan (W1–W4), staying a pure read-time projection — no capture, nothing hashed, `verify()` untouched.

- **Projection (`graph.ts`).** Each node now carries `rank` (role → vertical band: prompt 0 · step 1 · file/entity 2 · commit 3) and `turn` (first-appearance turn → horizontal time), and parallel identical relationships **merge into one weighted edge** instead of double-drawing. Purely additive layout hints; the exact combo-anchoring logic is unchanged. (A stray NUL separator in the edge key was replaced with a collision-proof `JSON.stringify([from,to,type])`.)
- **W1 — sharp.** The canvas buffer is **DPR-scaled** (retina-crisp; all hit-testing kept in CSS px), edges are **curved with direction arrowheads** and colour-by-type, labels get a **halo + level-of-detail** (only prompts/commits/risk/hubs at low zoom), and risk nodes get a **soft red glow** — the exfil chain is the screenshot.
- **W2 — hybrid layout.** **Barnes–Hut** repulsion (≈O(n log n)) + springs whose rest-length includes node radius (hubs stop crushing neighbours) + a **gentle time bias** (prompts drift up, outcomes down, earlier turns left). Seeded from `rank`/`turn`, it **settles by kinetic energy** (not a frame count) and **reheats** on interaction — it never sits frozen mid-tangle.
- **W3 — interaction.** **Drag-to-pin** a node (double-click unpins), **zoom-to-fit** (double-click empty / a `fit` button), **click a prompt to focus its turn**, hover tooltip, and full **keyboard** nav (arrows cycle, Enter drills in) with a focus ring.
- **W4 — fullscreen explorer.** An `⤢ fullscreen` overlay reusing the same model (layout continues seamlessly; the inline sim pauses) with a **minimap** (viewport rect + click-to-jump), **search** (fly-to + dim non-matches), and **type-filter chips**. One active sim at a time; every rAF/listener/overlay is torn down on close/switch.

**Verification.** Projection: 21 unit tests (17 prior + `rank`/`turn`/`weight`, earliest-turn anchoring, weighted-merge). Runtime: a **headless harness executes the shipped `CLIENT_JS`** and drives `mountGraph` against a fake retina canvas — proving it *runs* (not just parses), the sim **settles** (53 frames, no infinite spin), the **DPR fix** scales the buffer (900px→1800px @2×), and nodes/curves/halo-labels render with finite positions. `CLIENT_JS` still passes the validity + template-literal-safety gates. Full suite **215 pass**. Live: the daemon serves the new client; `/graph` returns `rank`/`turn`/`weight`; overview (prompt+file), detailed (prompt + 67 steps), and the exfil session (secret + red combo path anchored to its prompt) all render.

## Pivot — force-directed was unusable; rebuilt as a readable tree
Driven live (headless Chromium, retina) the force graph failed on real data: a session is mostly *disconnected* prompts with a handful of edges, so a physics layout had nothing to organise — prompts strung across the top, everything else floating. Verdict from the screenshots: unusable. The whole force/"Obsidian" renderer (Barnes–Hut, springs, minimap, drag-pin, DPR canvas physics) was **scrapped** and replaced with a deterministic **provenance tree** that answers the actual question — *what did this prompt produce?*

- **Projection (`graph.ts` → `buildTree`).** A rooted tree instead of a node-link graph. Overview: a `session` root whose children are one `prompt` node per turn; each prompt branches into the **files it changed · commits · subagents · risk** it produced. Single turn (`?prompt=`): that prompt is the root, fully expanded. Risk combos attach a `secret` (→ `host`) or `mcp` leaf to the exact turn that fired them (never fabricated). Pure read-time projection; nothing hashed; `verify()` untouched. `read-api.sessionTree` + the `/graph` route now return the tree.
- **Renderer (`mountTree`).** A deterministic tidy layout on a DPR-sharp canvas inside a scroll container — no physics, always the same shape. The **overview is a forest**: each turn is its own top-down subtree stacked vertically, so a busy session reads as a clean scrollable list (collapsed turns = one card each; a hidden-child count on the card). Expanding a turn drops its outputs below it. A single turn renders as the classic top-down tree (prompt → its outputs). Curved connectors, rounded node cards coloured by type, a red glow + red border on risk. Click a branch node to open/collapse; click a leaf to open its evidence dossier (reuses `insertDetail`). `fit` / `expand all`, and a `⤢ fullscreen` overlay with search (expands ancestors of matches + scroll-to).
- **Interaction is legible, not physical.** No dragging, no settling, no zoom-to-nothing — the previous failure modes are gone by construction.

**Verification.** Projection: 14 tree unit tests (session-root/forest shape, prompt→files/commits/subagents, exfil secret+host attached to the right turn, tool-poisoning mcp, no-fabrication when a combo anchors nowhere, lifecycle-turn filtering, distinct same-basename nodes, unknown-`?prompt` degrade, node counts + JSON round-trip). `CLIENT_JS` still passes the validity + template-literal-safety gates. Full suite **208 pass**. Live (headless Chromium @2×): the overview renders a scrollable list of turn cards (flagged turns red-bordered); a single turn renders `prompt → subagent + secret(exfil)` cleanly; `expand all` and the fullscreen explorer render the full forest — all razor-sharp and genuinely readable.

## Pivot 2 — from a tree to a re-traceable, deterministic **Sugiyama trace**
The tree read cleanly but wasn't a *forensic* instrument: the overview was a collapsed list, a single turn was a shallow fan, and it never answered the real question — *given this finding, what caused it and what did it cause?* This pass rebuilds the graph as a **layered causal DAG** (Sugiyama), with a **provenance trace as the default lens** rather than a whole-session dump. Still a pure read-time projection: no capture, nothing hashed, `verify()` byte-identical (checked live: `✓ chain intact` across 132 sessions / 76 signed checkpoints, unchanged).

**Why a layered DAG (not force, not a tree).** Our data is a DAG with a time axis — prompt → step → file/commit → finding. A force layout throws away causal direction and, fatally for evidence, is **non-deterministic** (the same session moves on every reload). A Sugiyama layout flows causality along one axis and is computed **server-side from a pure function**, so coordinates are byte-identical on every render (there is a determinism unit test).

**No d3-dag.** The daemon's CSP is `script-src 'unsafe-inline'` with *no* host allow-listed, so no external script loads at all — not a CDN, not a same-origin `/d3-dag.js` — and there is no build step. The Sugiyama layout is therefore **hand-rolled** in `graph.ts` (~250 lines): longest-path layering → dummy nodes for multi-layer edges → barycentre crossing reduction → a uniform-slot, bounded coordinate pass that centres each column (a linear trace draws as a straight left→right chain and can never diverge or overlap). It returns positioned nodes + routed edge points; the client only draws.

- **Projection (`graph.ts` → `buildTrace`).** `buildModel` assembles a sparse causal DAG — one `prompt` per turn; a `step` node **only** when it matters (a finding's antecedent/consequent, a risk-flagged step, or a subagent spawn) so ordinary edits attach their file straight to the prompt; `file`/`dir`/`commit` artifacts; `finding` + `host` for fired combos. A finding sits **between** its antecedent and consequent (`antecedent --flagged--> finding --flagged--> consequent`), so rooting there shows both what led to it and what it produced. `trace(root, depth)` takes the sub-DAG within *depth* causal hops (ancestry ∪ descendants); `layout` positions it. Anchoring stays exact — a combo that resolves to no visible turn is skipped, never fabricated.
- **Aggregation kills the star.** A parent's files are grouped by directory; any directory with **≥3** non-risk files collapses into one expandable `dir` node (*"9 files in src/"*), materialised into individual files on click (server re-lays-out with an `expand=` set). A risk-flagged file is never aggregated.
- **Visual encoding carries meaning.** Node fill/border/dot by **kind** (prompt·step·file·dir·commit·finding·host), size by significance tier, a red glow on risk; **edge stroke by relation** — `caused`/`wrote`/`read`/`committed`/`flagged`/`sent` differ in colour, width, dash, and arrowhead. Labels live inside non-overlapping cards (collision-free by construction) with the full text on hover.
- **Renderer (`mountDag`, SVG).** Crisp at any zoom, per-element hit-testing (no DPR canvas juggling). **Default = a trace rooted at the highest-severity finding** (else the most significant turn); controls are a **root selector** (findings first, then turns), a **depth 1/2/3/all** selector, a **whole-session** toggle (available, not default), fit, and fullscreen. **Click a node → re-root the trace there**; **double-click → its evidence dossier** (`insertDetail`); drag to pan, scroll to zoom, `fit` frames it (fits both axes for a small trace; centres on the root for a tall one). Re-root/depth/expand refetch the pure layout, so the view is always deterministic.
- **API.** `read-api.sessionTrace(store, id, {root, depth, whole, expand})`; daemon route `GET /api/session/:id/trace?root=&depth=&whole=&expand=` (the old `/graph` path still resolves to it).

**Verification.** 16 unit tests: whole-session prompt→file/commit edges, the exfil finding *between* antecedent and consequent (+host `sent`), default-roots-at-the-finding, depth bounds the causal radius, directory aggregation + expand, tool-poisoning server node, no-fabrication when a combo anchors nowhere, lifecycle-turn drop, default root = most-significant turn when there are no findings, root-selector ordering (findings first), every node within bounds + **edges only ever point to a later layer** (a proper DAG layering), **determinism** (whole-session *and* default trace lay out byte-identically across renders), coordinate stability regardless of combo order, JSON round-trip, empty-story. `CLIENT_JS` still passes the validity + template-literal-safety gates. Full suite **210 pass**, `verify()` unchanged. Live (headless Chromium @2×): the exfil session opens on `turn 1 → Read ~/.ssh → [EXFIL] → send`, reading left→right; the build session's default trace fans `turn 51` into files + risk-Bash steps + `9 files in src/` · `4 files in test/` aggregates; clicking a node re-roots, clicking a dir expands (10→18 nodes), and the whole-session view renders layered — not a hairball.
